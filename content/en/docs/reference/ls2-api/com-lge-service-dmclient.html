getStatus

ACG: softwareupdate.query
  • Added: API level 27

Description

Gets all messages from FOTA service such as result of server checking, download progress, and the status of service.

Note: This method has to be subscribed.

Parameters

Name

Required

Type

Description
subscribe Optional Boolean

Indicates if subscribed to get notifications. Possible values are:

  • true: (default) Subscribed for notifications.
  • false: Not subscribed.

Call Returns

Name

Required

Type

Description
subscribed Required Boolean

Indicates if subscribed to get notifications. Possible values are:

  • true: Subscribed for notifications.
  • false: Not subscribed.
status Required String

Indicates the status of dmclient (FOTA service). Possible values are:

  • DM_COMMUNICATING: Service is communicating with the server.
  • DM_COMPLETE: Communication with the server is completed and there is no package to download.
  • DL_DD_RECEIVED: Communication with the server is completed and there is a package to download.
  • DL_OBJ_DOWNLOADING: Service is downloading the package from the server.
  • DL_OBJ_DOWNLOADED: Downloading from the server is completed.
  • FOTA_RESULT_SUCCESS: Update is successful.
  • FOTA_RESULT_FAIL: Update has failed.
extData Optional String

Target version info or SW update error info.
progress Optional Number

Indicates current progress, when the status is DL_OBJ_DOWNLOADING. The value is a percentage between 0 and 100.

For example:

  • 0: Indicates 0% completion.
  • 100: Indicates 100% completion.
mdata Optional Number

Information about SW update initiation factors. Possible values are:

  • 4: User request
  • 5: Device Request
returnValue Required Boolean

Indicates the status of the operation. Possible values are:

  • true: Indicates that the operation was successful.
  • false: Indicates that the operation failed.

Subscription Returns

Name

Required

Type

Description
subscribed Required Boolean

Indicates if subscribed to get notifications.
status Required String

Indicates the status of dmclient (FOTA service).
extData Optional String

Target version info or SW update error info.
progress Optional Number

Indicates the current progress, when the status is DL_OBJ_DOWNLOADING. 
mdata Optional Number

Information about SW update initiation factors.
returnValue Required Boolean

Indicates the status of the operation.

Example

Example : WIth progress returned

# luna-send -n 100 -f luna://com.lge.service.dmclient/getStatus '{ "subscribe":true }'

Response:

{
    "subscribed": true,
    "status": "DL_OBJ_DOWNLOADING",
    "returnValue": true,
    "progress": 15
}

Example : WIth mdata returned

# luna-send -n 100 -f luna://com.lge.service.dmclient/getStatus '{ "subscribe":true }'

Response:

{
    "subscribed": true,
    "status": "DM_COMPLETE",
    "returnValue": true,
    "mData": 4
}
